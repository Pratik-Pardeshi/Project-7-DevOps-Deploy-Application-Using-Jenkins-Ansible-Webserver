Project 7: Jenkins Ansible Intigration with SSH

Automate, integrate, and deploy seamlessly with Jenkins, Ansible, Web-Server, Developer Server and AWS EC2.

Discover More

Project Overview

This DevOps CI/CD project automates the deployment pipeline using Jenkins, Ansible,Web server, Developer Server and AWS EC2 instances. The setup ensures a seamless integration and delivery process, streamlining code deployment and updates.

Project Components

  • EC2 Instances:
    • Jenkins-Server: Manages CI/CD pipelines and automates builds.
    • Ansible-Server: Executes playbooks to deploy code to the web server.
    • Web-Server: Hosts the deployed application or website.
    • Developer: Where code changes are made and pushed to GitHub.
  • Tools Used:
    • Jenkins: Automates building, testing, and deploying code.
    • Ansible: Manages configuration and deployment tasks.
    • GitHub: Hosts the code repository and triggers Jenkins builds.
  • Deployment Process:
    • Code is pushed to GitHub by the developer.
    • Jenkins detects the push, builds the code, and prepares deployment artifacts.
    • Jenkins sends the artifacts to the Ansible server.
    • Ansible executes a playbook to deploy the code to the web server.

Detailed Steps

  1. Create EC2 Instances: Set up four EC2 instances for Jenkins, Ansible, Web, and Developer roles.
  2. Install Jenkins: On the Jenkins server, install Java, Jenkins, and configure it to manage builds.
  3. Install Ansible: On the Ansible server, install Ansible and configure it to manage deployments.
  4. Set Up Passwordless SSH: Configure SSH keys for passwordless connections between Jenkins, Ansible, and the Web server.
  5. Create Ansible Playbook: Develop a playbook to deploy the code to the web server.
  6. Integrate with GitHub: Set up GitHub webhooks to trigger Jenkins jobs on code changes.
  7. Configure Jenkins Job: Set up Jenkins to build the code, send artifacts to Ansible, and run the playbook.
  8. Deploy and Test: Ensure the deployment pipeline works end-to-end by pushing code changes and verifying updates on the web server.
